## Problem

Build #4962 failing on `dagger-knife-lint-plus-typecheck-plus-test` (hard fail).

ESLint errors in `packages/scout-for-lol/packages/app/src/components/report-query-editor.tsx`:

- Line 36: `@typescript-eslint/no-unsafe-assignment` — `monacoRef.current = monaco;`
- Line 37: `@typescript-eslint/no-unsafe-argument` — `registerScoutQlLanguage(monaco);`

Root cause: monaco-editor 0.55 changed `editor.api.d.ts` structure (mixed ambient
declarations + module exports). `@typescript-eslint` marks the `monaco` parameter
inferred from `@monaco-editor/react`'s `OnMount` type as "error typed" because
`typeof import('monaco-editor/esm/vs/editor/editor.api')` resolution fails at the
lint layer. TypeScript itself (`tsc --noEmit`) passes with `skipLibCheck: true`.

## Fix

In `packages/scout-for-lol/packages/app/src/components/report-query-editor.tsx`:

- Removed `OnMount` import from `@monaco-editor/react`
- Explicitly typed `handleMount` parameters as `Monaco.editor.IStandaloneCodeEditor`
  and `typeof Monaco` (where `Monaco = import type * as Monaco from "monaco-editor"`)
- This bypasses the "error typed" inference from `OnMount`'s parameter types
