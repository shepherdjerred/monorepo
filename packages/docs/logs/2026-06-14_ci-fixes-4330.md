# Main CI fixes — build #4330

## Symptoms

Main build [#4330](https://buildkite.com/sjerred/monorepo/builds/4330) (commit `1439c18cf`, the rss-stylesheet rename) had three failed jobs:

| Job                                | Soft? | Cause                                                              |
| ---------------------------------- | ----- | ------------------------------------------------------------------ |
| `:scissors: Knip`                  | yes   | Known soft-failure being hardened — ignore.                        |
| `:terraform: Apply Tailscale ACLs` | no    | `tofu init` flaked on a slow GitHub release-CDN provider download. |
| `:bookmark: Version Commit-Back`   | no    | Auto-merge call used `--rebase`; repo only allows squash.          |

## Apply Tailscale ACLs

Inside the `tailscale` stack, `tofu init -input=false` failed downloading the provider SHA256SUMS:

```
Error while installing tailscale/tailscale v0.29.2:
  Get "https://github.com/tailscale/terraform-provider-tailscale/releases/download/v0.29.2/.../SHA256SUMS":
  context deadline exceeded
```

`tofu init` only retries the registry call twice, so a single slow CDN response fails the entire stack. Fix in `.dagger/src/release.ts`: wrap `tofu init` in a bounded 5-attempt shell retry with linear backoff (5s/10s/15s/20s/25s), reused by both `tofuApplyHelper` and `tofuPlanHelper`. Hard-fails if all attempts miss.

## Version Commit-Back

The job opened [PR #1232](https://github.com/shepherdjerred/monorepo/pull/1232) correctly, then tried to enable auto-merge and got:

```
GraphQL: Merge method rebase merging is not allowed on this repository (enablePullRequestAutoMerge)
```

`packages/homelab/src/tofu/github/repos.tf` only enables `allow_squash_merge=true` on `shepherdjerred/monorepo` — `allow_rebase_merge=false`, `allow_merge_commit=false`. Three call sites in `.dagger/src/release.ts` were hard-coded to `--rebase`, and the hygiene test (`scripts/ci/src/__tests__/dagger-hygiene.test.ts`) was asserting `--rebase` while forbidding `--squash`, which is how the bad config got through review.

Fix:

- `.dagger/src/release.ts:939` `versionCommitBackHelper` → `--squash`.
- `.dagger/src/release.ts:998` `ciBaseVersionCommitBackHelper` → `--squash`.
- `.dagger/src/release.ts:1068` `cooklangVersionCommitBackHelper` → `--squash`.
- `scripts/ci/src/__tests__/dagger-hygiene.test.ts` → assert `--squash`, forbid `--rebase` and `--merge` (with a comment pointing back to `repos.tf` so the next reviewer sees the ground truth).

Since the squash result of a one-commit bump PR is byte-identical to a rebase, no behaviour change beyond making auto-merge actually fire.

### Caveats

- The retry only protects against transient registry/CDN timeouts. A persistent outage will still surface as `exit 1` after ~75 seconds plus 5 init attempts (~couple of minutes total worst case), which is the intent.
- PR #1232 (the bump PR that was stuck open from build #4330) is still open. After main lands, the next `Version Commit-Back` run will re-rebase the bump branch and call `--auto --squash` on it, which will satisfy auto-merge and close it.
