## CI pipeline & gates

| #   | Recurring symptom                                                                                                                                                                                                                        | Why it costs time                                                                                                                        | Proposed fix                                                                                                                                    |
| --- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Any `.dagger/src/` change triggers a full **"build everything"** (all docker images, ~30–60 min).                                                                                                                                        | Every cherry-pick/merge that touches `.dagger` (constant this session) re-runs the entire image matrix; cascades on each conflict merge. | Content-hash each image's transitive input closure; rebuild only images whose inputs actually changed.                                          |
| 2   | Soft-fail jobs (knip/trivy/large-file/semgrep) flip the aggregate `buildkite/monorepo/pr` status to **FAILURE** even when every hard gate passed.                                                                                        | Every PR with a soft finding reads "red"; had to drill into each build all session to confirm it's actually fine.                        | Don't let `soft_failed` jobs flip the aggregate; publish a separate hard-gates rollup as the merge signal.                                      |
| 3   | The Greptile gate (`wait-for-greptile.ts`) **can't pass on a no-code PR** — docs-only / empty-diff PRs time out after 1200s ("review check: not started").                                                                               | Blocks any docs/markdown PR with no recourse but admin-merge (#1153).                                                                    | Treat "no reviewable files / Greptile never picked up the commit" as pass/skip instead of a 1200s timeout-failure.                              |
| 4   | A step's **soft-vs-blocking status is easy to mis-wire and the mismatch is silent** — a step can sit in `blockingGates` while a test/comment claims `ASYNC_OR_SOFT_CI` (or vice-versa). Bit react-version-sync (#1156) AND knip (#1151). | The lefthook↔CI parity test doesn't fully prevent it; each mismatch is a separate debugging session.                                     | One source of truth per step for soft/blocking, asserted end-to-end (gate → annotation → soft_fail flag).                                       |
| 5   | Scripts that **shell out to a tool absent from the Dagger image fail silently** — `check-todos.ts` used `rg`, not in `bunQualityBase`; `rg`-not-found exits 1 = same as `rg`-no-matches, so the gate ran vacuously / false-flagged.      | A correctness gate that silently doesn't work; real violations would slip through.                                                       | `command -v <tool>` guard + hard-fail with a clear message in any script that shells out; audit the quality image for assumed tools (rg, etc.). |

## Dependency / merge hygiene

| #   | Recurring symptom                                                                                                                                                                                                                                                                        | Why it costs time                                                                                        | Proposed fix                                                                                                                                            |
| --- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 7   | **Lockfile drift is only caught in a deep CI job** — `bun install --frozen-lockfile` fails when a committed `bun.lock` is stale vs `package.json` (hit #875, #1151). The lefthook `lockfile-check` didn't catch these.                                                                   | A late, cryptic CI failure for something a pre-commit gate should catch.                                 | A reliable per-changed-package frozen-install pre-commit gate (verify why the current `lockfile-check` misses it).                                      |
| 8   | A new intra-repo `file:` dep must be **hand-wired into `.dagger/src/deps.ts` `WORKSPACE_DEPS`** or consumers fail `bun install` in their per-package Dagger job — easy to forget (bit `discord-stream-lifecycle` on #1146 AND #1152 independently).                                      | Each omission is a deep-docker `Could not find package.json for file:…` failure.                         | Check that every intra-repo dep in a package.json has a matching `WORKSPACE_DEPS` edge; fail fast at lint.                                              |
| 9   | **Git auto-merge silently regresses shared allow/exclude lists** — non-conflicting hunks resolve to the branch's older version, dropping main-side additions (hit `scripts/check-suppressions.ts` + `lefthook.yml` excludes twice: #875, #1045). No conflict markers; fails later in CI. | A "clean" merge silently reverts shared config; surfaces as an unrelated suppression/shellcheck failure. | Keep these allow/exclude lists in append-only data files; and/or post-merge diff shared-config vs `origin/main` and re-apply dropped main-side entries. |
| 10  | **Hot shared files re-conflict on every sibling merge** — `versions.ts`, `catalog.ts`, mcp-gateway `config.json`/`index.ts`, `.dagger/src/deps.ts`, temporal `bun.lock`, `controller-ui.tsx`. Whack-a-mole all session: resolve, sibling merges, re-conflict.                            | Same files re-merged repeatedly across #1044/#1095/#1151/#1152/#1045.                                    | Generate `versions.ts`/`catalog.ts` from data files; split per-domain to shrink the conflict surface.                                                   |
