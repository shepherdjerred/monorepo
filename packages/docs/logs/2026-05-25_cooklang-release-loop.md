# Cooklang Release Loop

## Summary

Main Buildkite was looping through Cooklang version bumps. Build `#2944` ran `cooklang-publish`, published external plugin `1.0.10 -> 1.0.11`, opened monorepo PR `#939`, and the auto-merged bump triggered build `#2950`. Build `#2950` still had `cooklang-publish` waiting when inspected, so it was canceled before it could publish `1.0.12`.

Root cause: `scripts/ci/src/pipeline-builder.ts` already skipped `version-commit-back` on `affected.isAutoGenerated`, but Cooklang release did not use that guard. A bot-authored Cooklang manifest bump therefore counted as `cooklangChanged`, scheduled `cooklang-release`, published the next patch, opened another PR, and repeated.

Fix: suppress `cooklangReleaseGroup` when `affected.isAutoGenerated` is true. Real non-generated Cooklang changes and normal full release builds still schedule the Cooklang release step.
