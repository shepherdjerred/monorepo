- Multiple tailnets for a single organization (alpha) lets admins manage more than one tailnet under a shared organization, using a common identity provider and domain
- This is useful for separating production and staging environments, different teams, or different security zones while keeping centralized identity management
- All tailnets in the organization share the same admin console organization context, so you can switch between them without separate logins
